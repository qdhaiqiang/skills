# 周报模板结构参考

生成周报时，Agent 应根据扫描数据按以下结构填充内容。

## 模板模块

### 一、目标与背景
- **本周目标与背景**：总结本周做了什么、为什么做。从各项目的会话标题中提取关键任务。
- **本周最重要的目标**：2-3 个最高优先级目标，含完成标准。

### 二、完成情况
- 表格形式，列：工作项 | 状态 | 证据/结果 | 备注
- 状态用 ✅完成 / 🔄部分完成 / ❌未开始
- 从会话标题中提取工作项名称

### 三、过程证据
- 列出证据类型清单（PR、截图、日志、会议纪要等）
- 关键证据说明（用高亮块）

### 四、AI 协作
- AI 协作总览表：任务类型 | 使用场景 | 最终结论
- 人工做了什么：输入 → 输出 → 审查 → 采纳/拒绝
- 最值得讲的 2 件事

### 五、价值与指标
- 表格：指标项 | 上周/基准 | 本周结果 | 下周目标 | 说明

### 六、风险与问题
- 表格：风险/问题 | 状态 | 影响 | 缓解动作 | 责任人/截止时间
- 状态用 🔴打开 / 🟡缓解中 / 🟢关闭

### 七、下周计划（按天输出）

**必须按天列出**，每天包含工作内容和验收标准。未排满 5 天时说明剩余天数用途。

表格格式：
```
| 日期 | 工作内容 | 验收标准 |
|------|----------|----------|
| 周一 | 定制下单—面料列表接口对接、款式配置接口对接 | 面料数据从后端加载，可切换款式、看到价格变化 |
| 周二 | 购物车—CRUD 接口对接，批量操作 | 购物车增删改查、批量提交均返回正确状态码 |
| 周三 | 快捷下单—编号搜索 + 快速下单接口 | 输入款式编号+面料编号可完成快速下单 |
| 周四 | 面料库—列表/筛选/搜索接口对接 | 面料数据从后端加载，支持品牌、类别筛选 |
| 周五 | 多语言—后端字典接口对接，五语切换测试 | 切换语言后页面文案和数据均对应显示 |
```

### 八、复盘与自检
- 做得好 2-3 条
- 做得不好 2-3 条
- 下周改变什么
- 提交前自检 checklist

## HTML 生成规范

- 蓝白主题（blue-50~blue-900）
- 白色卡片式布局，蓝色渐变封面
- 表格：蓝灰色表头，悬停高亮
- 状态徽章：绿/黄/红三色
- 任务卡片：四宫格浅蓝底色
- 下周计划按天表格
- 响应式 + 打印友好
