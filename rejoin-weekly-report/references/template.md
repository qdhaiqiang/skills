# 周报模板结构参考

生成周报时，Agent 应根据扫描数据按以下结构填充内容，与 PPTX 模板保持一致。

## 模板模块

### 一、目标与背景
- **本周目标与背景**：总结这周为什么做这些事（客户痛点 / 内部目标 / 项目节点），不要只写"做了 X 功能"
- **本周最重要的 1-3 个目标**：含完成标准
- **判断完成的标准**：怎么叫完成？由谁确认？看什么结果？

### 二、完成情况
- 表格列：工作项 | 状态 | 证据/结果 | 备注
- 状态用 ✅完成 / 🔄部分完成 / ❌未开始
- "已完成"必须带证据；"未完成"必须写原因和后续动作
- 如果工作项很多，保留 3-5 条最关键的

### 三、过程证据
- **证据类型清单**：至少覆盖 2 类不同证据（页面 + PR / 日志 + 会议纪要）
- **证据截图区**：每张图下面补一句"它证明了什么"
- 证据图 1：页面效果 / 系统截图
- 证据图 2：PR / 测试 / 构建日志
- 证据图 3：会议纪要 / 客户反馈 / 数据看板

### 四、AI 协作
- **AI 协作总览**：任务类型 | 使用场景/为什么用 AI | 最终结论
- **人工审查说明**：输入 → 输出 → 审查 → 采纳/拒绝
- **最值得讲的 2 件事**：最关键的使用 + 最能体现人工审查的使用

### 五、价值与指标
- 表格列：指标项 | 上周/基准 | 本周结果 | 下周目标 | 说明
- 尽量写变化而非静态数字
- 无法量化时用"数量 + 状态"组合

### 六、风险与问题
- 表格列：风险/问题 | 状态 | 影响 | 缓解动作 | 责任人/截止时间
- 状态用 🔴打开 / 🟡缓解中 / 🟢关闭
- 必须有状态、影响、动作、责任人
- 已关闭的问题也可保留 1 条证明闭环

### 七、下周计划（按天输出）

**必须按天列出**，每天包含工作内容、验收标准、负责人。未排满 5 天时说明剩余天数用途。

表格格式：
```
| 日期 | 工作内容 | 验收标准 | 负责人 |
|------|----------|----------|--------|
| 周一 | 定制下单—面料列表接口对接、款式配置接口对接 | 面料数据从后端加载，可切换款式、看到价格变化 | 张三 |
| 周二 | 购物车—CRUD 接口对接，批量操作 | 购物车增删改查、批量提交均返回正确状态码 | 张三 |
| 周三 | 快捷下单—编号搜索 + 快速下单接口 | 输入款式编号+面料编号可完成快速下单 | 李四 |
| 周四 | 面料库—列表/筛选/搜索接口对接 | 面料数据从后端加载，支持品牌、类别筛选 | 李四 |
| 周五 | 多语言—后端字典接口对接，五语切换测试 | 切换语言后页面文案和数据均对应显示 | 张三 |
```

额外内容：
- **需要谁支持**：数据 owner / 业务 owner / QA / 研发同学
- **需要什么决策**：需要团队确认的事项

### 八、复盘与自检
- **做得好**：继续保持的 2-3 点
- **做得不好**：需要修正的 2-3 点
- **下周改变什么**：把复盘转成动作（问题 → 原因 → 改进 → 验证）
- **提交前自检**：checklist

## HTML 生成规范

- 页面内容区域**最大宽度 1200px**，居中对齐
- 蓝白主题（blue-50 ~ blue-900）
- 白色卡片式布局，蓝色渐变封面
- 表格：蓝灰色表头，悬停高亮
- 状态徽章：绿（✅完成）/ 黄（🔄部分完成）/ 红（❌未开始 / 🔴打开）
- 证据区：灰色虚线边框占位框，标注"贴图位置"
- 下周计划按天表格，含"负责人"列
- 响应式 + 打印友好
