# Rejoin 周报生成器

从 rejoin 的 SQLite 数据库中直接查询所有 Agent 会话，按项目归类汇总，生成符合公司周报模板的 Markdown 文档和蓝白主题 HTML 网页。

## 工作流程

1. 运行扫描脚本，从 rejoin 数据库中提取会话数据
2. 过滤掉指定项目（由用户通过 --exclude 指定）
3. 按项目分组、分析会话标题和内容
4. 生成 Markdown 周报，再转换为 HTML

## 使用方式

### 步骤 1：扫描并生成原始数据

```bash
python3 scripts/scan_sessions.py \
  --output /tmp/rejoin_weekly.json
```

参数：
- `--output`：输出 JSON 文件路径（必填）
- `--exclude`：要排除的项目名关键字，可多次指定（可选）
- `--since`：起始日期 `YYYY-MM-DD`（可选，默认本周一）
- `--db`：rejoin 数据库路径（可选，默认 `~/.local/share/rejoin/index.db`）

### 步骤 2：生成 Markdown 周报

```bash
python3 scripts/generate_report.py \
  --input /tmp/rejoin_weekly.json \
  --output /path/to/周报.md
```

参数：
- `--input`：步骤 1 输出的 JSON 文件（必填）
- `--output`：输出的 Markdown 文件路径（必填）
- `--reporter`：汇报人姓名/角色（可选，默认使用本机计算机名）
- `--week`：周数（可选，默认计算当前是第几周）
- `--title`：汇报标题（可选，默认"青岛红创 第 N 周工作展示汇报"）

### 步骤 3：Agent 填充内容（含按天工作计划）

报告框架生成后，Agent 会根据扫描结果填入各模块内容。**特别注意"下周计划"应按天输出**，每天包含：
- 工作内容（要做什么、涉及哪些模块）
- 验收标准（怎么叫完成、可验证的结果）

示例格式：

```
| 日期 | 工作内容 | 验收标准 | 负责人 |
|------|----------|----------|--------|
| 周一 | 定制下单—面料列表接口对接 | 面料数据从后端加载，支持筛选搜索 | 张三 |
| 周二 | 购物车—CRUD 接口对接 | 增删改查、批量提交均正常 | 张三 |
| 周三 | 管理中心—门店信息接口对接 | 门店资料可查看和编辑 | 李四 |
| ... | ... | ... | ... |
```

### 步骤 4：生成 HTML（在 Agent 中）

报告生成后，请 Agent 将 Markdown 转换为蓝白主题 HTML：

```
请将 {output}.md 的内容转换为蓝白主题的 HTML 网页，保存为 {output}.html
要求：
- 页面内容区域最大宽度 1200px，居中对齐
- 蓝白主题（blue-50 ~ blue-900）
- 白色卡片式布局，蓝色渐变封面
- 表格：蓝灰色表头，悬停高亮
- 状态徽章：绿（✅完成）/ 黄（🔄部分完成）/ 红（❌未开始 / 🔴打开）
- 下周计划按天表格，含"负责人"列
- 响应式 + 打印友好
```

## 周报模板结构

生成的报告包含 8 个标准模块：
1. 目标与背景
2. 完成情况（表格）
3. 过程证据
4. AI 协作
5. 价值与指标
6. 风险与问题
7. 下周计划（按天输出，含工作内容和验收标准）
8. 复盘与自检

## 注意事项

- 脚本只做数据提取和初步归类，不会自动填充所有模板字段
- 目标、风险、下周计划（按天）等需要人工判断的内容，由 Agent 根据扫描结果推断填充
- **下周计划必须按天列出**，未排满 5 天时说明剩余天数安排
- rejoin 服务需要在运行时保持数据库更新（正常启动即可，脚本直读 SQLite 不依赖 API）
